# DotBoxD Performance Findings

This file tracks the focused performance findings from the current perf-hunter pass.
All numbers are local Release stopwatch probes on the same machine and are intended
as targeted before/after evidence, not BenchmarkDotNet statistical reports.

## Results

| Finding | Probe | Workload | Before total | Before ns/op | Before alloc | Before B/op | After total | After ns/op | After alloc | After B/op | Notes |
| --- | --- | --- | ---: | ---: | ---: | ---: | ---: | ---: | ---: | ---: | --- |

| HTTP response metadata was measured once for quota checking and again for charging. | `--probe-http-metadata` | 100,000 responses with 24 headers | 615.4 ms | 6,154 | 354,700,000 B | 3,547.0 | 99.8 ms | 998 | 176,800,040 B | 1,768.0 | Same `55,300,000` charged network bytes. |

| Capability-gated dispatch/bindings resolved the same grant twice. | `--probe-capability-grant-lookup` | 1,000,000 `RequireCapability` + `GetCapability` pairs | 24.5 ms | 24.5 | 728 B | ~0 | 2.2 ms | 2.2 | 728 B | ~0 | Time-only improvement; grant cache is keyed by capability id and `EffectiveGrantClock`. |
| Structural compiled binding validation materialized `SandboxValue.Type` for nested list/map/record arguments. | `--probe-compiled-binding-structural-validation` | 1,000,000 list + record argument-pair validations (2,000,000 checks) | 350.2 ms | 175.1 | 520,000,040 B | 260.0 | 74.8 ms | 37.4 | 40 B | ~0 | Probe compares the legacy `.Type.Equals` shape check with the direct matcher now used by the dispatcher. |
| Compiled `list.add` materialized `item.Type` for structural item checks. | `--probe-list-add-type-match` | 1,000,000 nested record item type checks | 236.9 ms | 236.9 | 312,000,040 B | 312.0 | 83.0 ms | 83.0 | 40 B | ~0 | Exact non-allocating matcher preserves standalone type-shape checking without building record `SandboxType` graphs. |

| Compiled I32 math intrinsics used boxed direct binding helpers inside raw loops. | `--probe-i32-math-intrinsic` | 1,000,000 charged `math.abs` calls | 7.5 ms | 7.5 | 11,643,616 B | 11.6 | 3.9 ms | 3.9 | 40 B | ~0 | Raw helpers preserve binding host-call/fuel accounting and avoid `SandboxValue` argument/result materialization. |
| Non-loop compiled I32 math intrinsic assignments used boxed direct binding helpers. | `--probe-i32-math-intrinsic` | 1,000,000 charged `math.abs` calls | 7.5 ms | 7.5 | 11,643,616 B | 11.6 | 3.1 ms | 3.1 | 40 B | ~0 | The raw emitter now covers assignment/raw-I32 consumers outside loop fast paths; direct returns still box the final value. |
| Non-loop F64 `math.floor`/`ceil`/`round` assignments missed the raw helper path. | `--probe-f64-math-intrinsic` | 1,000,000 charged `math.floor` calls | 9.4 ms | 9.4 | 48,000,040 B | 48.0 | 2.9 ms | 2.9 | 40 B | ~0 | Raw helpers preserve binding host-call/fuel accounting and avoid boxed F64 argument/result materialization. |
| Direct-return F64 math intrinsics missed the raw helper path. | `--probe-f64-math-intrinsic` | 1,000,000 charged `return math.floor(...)` shapes | 9.4 ms | 9.4 | 48,000,040 B | 48.0 | 5.9 ms | 5.9 | 24,000,040 B | 24.0 | Return path still boxes the final `F64Value`; the raw helper removes the boxed argument side. |
| General unary negation compiled `I64`/`F64` through boxed runtime helpers. | `--probe-raw-unary-negation` | 1,000,000 assign-style unary negations | 9.4 ms / 5.2 ms | 9.4 / 5.2 | 48,000,040 B / 48,000,040 B | 48.0 / 48.0 | 0.9 ms / 0.7 ms | 0.9 / 0.7 | 40 B / 40 B | ~0 / ~0 | `I64` / `F64`; return-style final boxing measured 5.2 ms / 4.1 ms and 24.0 B/op. |
| Numeric conversion assignments boxed converted values, then unboxed into typed locals. | `--probe-numeric-conversion` | 1,000,000 assign-style numeric conversions | 5.6 ms / 3.8 ms | 5.6 / 3.8 | 24,000,040 B / 24,000,040 B | 24.0 / 24.0 | 0.6 ms / 0.7 ms | 0.6 / 0.7 | 40 B / 40 B | ~0 / ~0 | `I32->I64` / `I64->F64`; raw `Conv_I8`/`Conv_R8` emission is limited to assignment/raw-primitive consumers, and direct returns still box the final value. |
| Verifier opcode validation built branch-target sets for branch-free methods. | `--probe-verifier-opcode-branches` | 5,000 branch-free offset scans over 10,000 instruction offsets | 553.6 ms | 110,720 | 2,693,440,040 B | 538,688.0 | 11.2 ms | 2,240 | 40 B | ~0 | Branch target state is now allocated only when a branch/switch target is actually observed; branchy methods still validate against the same offset set. |
| Scalar literal safety checks used stack-backed flatten walks even for non-collection literals. | `--probe-literal-scalar-safety` | 1,000,000 I32 `ContainsDangerousReference` + `Validate` pairs | 169.8 ms | 169.8 | 304,000,040 B | 304.0 | 27.9 ms | 27.9 | 40 B | ~0 | Probe compares the legacy flatten-based scalar walks with the direct scalar checks now used before collection fallback. |
| Type validation checked `IsForbidden` after `IsKnown`, repeating the structural walk. | `--probe-sandbox-type-validation` | 1,000,000 nested `SandboxType.IsKnown` validation checks | 532.4 ms | 532.4 | 40 B | ~0 | 294.4 ms | 294.4 | 40 B | ~0 | `IsKnown` already rejects forbidden names recursively; diagnostics remain on the same unknown-or-forbidden branches. |
| Registered server-extension service lookup created a fresh `DispatchProxy` on every call. | `--probe-server-extension-proxy-lookup` | 1,000,000 `ServerExtension<TService>` lookups | 321.7 ms | 321.7 | 288,002,456 B | 288.0 | 22.3 ms | 22.3 | 80 B | ~0 | Probe simulates the old lookup with direct `ServerExtensionProxy.Create`; cached registrations are invalidated on unregister/replacement. |
| Installed server-extension invocation scanned module functions to resolve the same RPC entrypoint on every call. | `--probe-installed-rpc-input` | 200,000 RPC input builds over 512 module functions | 350.9 ms | 1,754.5 | 6,400,040 B | 32.0 | 1.4 ms | 7.0 | 40 B | ~0 | Cached the resolved `SandboxFunction` and caller argument count on `InstalledKernel` construction. |
